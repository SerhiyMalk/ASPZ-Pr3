# ASPZ-Pr3
### Завдання 3.1: Експерименти з лімітом відкритих файлів
* **Опис:** Дослідження роботи команди `ulimit` для керування кількістю одночасно відкритих файлових дескрипторів.
* **Результат:** Проведено експерименти з м'якими (`ulimit -aS`) та жорсткими (`ulimit -aH`) лімітами. Перевірено зміну ліміту (`ulimit -n <значення>`) та проаналізовано різницю в правах між звичайним користувачем та `root`.

# Завдання 3.2
* **Опис:** Дослідження примусового завершення процесу при вичерпанні ліміту процесорного часу.
* **Результат:** Встановлено жорсткий ліміт на час виконання (`ulimit -t 2`). За допомогою утиліти `time` та команди `md5sum /dev/zero` зафіксовано аварійне завершення процесу системою (сигнал `SIGKILL` / статус `Killed`) через обмеження безпеки середовища Docker.

# Завдання 3.3
* **Опис:** Програма на C, яка генерує кидки кубика та записує їх у файл із жорстким обмеженням на розмір.
* **Результат:** Використано функцію `setrlimit` для встановлення ліміту `RLIMIT_FSIZE` (50 байт). Реалізовано коректне перехоплення сигналу `SIGXFSZ`, щоб програма завершувалася безпечно, а не аварійно.

# Завдання 3.4
* **Опис:** Програма, що імітує лотерею (вибір 7 із 49 та 6 із 36 чисел) у нескінченному циклі з лімітом на використання ЦП.
* **Результат:** Встановлено ліміт часу процесора в 1 секунду (`RLIMIT_CPU`). Програма успішно перехоплює сигнал ядра `SIGXCPU` та виводить повідомлення про коректне завершення роботи замість стандартного аварійного падіння.

# Завдання 3.5
* **Опис:** Програма для копіювання вмісту з одного файлу в інший з обробкою аргументів та системних лімітів.
* **Результат:** Реалізовано перевірку наявності двох аргументів (`argc`), перевірку прав доступу на читання (`r`) та запис (`w`), а також обробку ситуації перевищення максимального розміру файлу (перехоплення `SIGXFSZ`).

# Завдання 3.6
* **Опис:** Перевірка впливу ліміту сегмента стека на роботу рекурсивних алгоритмів.
* **Результат:** Створено програму з глибокою рекурсією, що виділяє пам'ять у стеку (локальні масиви). Практично доведено, що зменшення системного ліміту стека (через команду `ulimit -s 1024`) пропорційно зменшує максимальну глибину рекурсії до моменту виникнення `Segmentation fault`.
